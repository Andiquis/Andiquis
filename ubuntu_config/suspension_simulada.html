🌘 Brillo automático por inactividad (Ubuntu + Wayland)

Objetivo: que Ubuntu nunca se suspenda, pero que el brillo baje casi a cero cuando no hay actividad y vuelva instantáneamente al mover mouse o teclado.

---

🧠 Idea general

  • No se usa suspensión
  • No se usa bloqueo
  • Wayland no mide tiempo exacto, solo activo / inactivo
  • GNOME decide cuándo estás inactivo
  • Un script controla el brillo
  • Un servicio de usuario lo ejecuta siempre
---

📦 Paquetes necesarios

sudo apt install brightnessctl
---

📜 Script de brillo

Archivo:

~/.local/bin/brillo_inactividad.sh
#!/bin/bash

BRILLO_MIN=1
apagado=false

SESSION_ID=$(loginctl list-sessions --no-legend | awk -v user="$USER" '$3 == user {print $1; exit}')
BRILLO_NORMAL=$(brightnessctl -d amdgpu_bl1 get)

while true; do
    idle=$(loginctl show-session "$SESSION_ID" -p IdleHint --value 2>/dev/null)

    if [ "$idle" = "yes" ] && [ "$apagado" = false ]; then
        brightnessctl -d amdgpu_bl1 set $BRILLO_MIN
        apagado=true
    fi

    if [ "$idle" = "no" ] && [ "$apagado" = true ]; then
        brightnessctl -d amdgpu_bl1 set $BRILLO_NORMAL
        apagado=false
    fi

    sleep 2
done

Dar permisos:

chmod +x ~/.local/bin/brillo_inactividad.sh
---

⚙️ Servicio systemd (usuario)

Archivo:

~/.config/systemd/user/brillo-inactividad.service
[Unit]
Description=Brillo automático por inactividad (Wayland)

[Service]
ExecStart=/home/andi/.local/bin/brillo_inactividad.sh
Restart=always

[Install]
WantedBy=default.target

Activar (una sola vez):

systemctl --user daemon-reexec
systemctl --user enable brillo-inactividad.service
systemctl --user start brillo-inactividad.service
---

🔍 Comprobar estado

systemctl --user status brillo-inactividad.service

Debe mostrar: Active: active (running)

---

⏱️ Control del tiempo de inactividad

Ver tiempo actual:

gsettings get org.gnome.desktop.session idle-delay

Ejemplo: 3 minutos

gsettings set org.gnome.desktop.session idle-delay 180

GNOME decide cuándo estás inactivo.
El script solo obedece.

---

🧪 Comportamiento final

  • Inactividad → brillo casi a cero
  • Movimiento → brillo vuelve
  • Reinicias → sigue funcionando
  • No se suspende
  • No se bloquea
  • No depende de terminal
